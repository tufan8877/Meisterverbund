Baue eine vollständige Web-App namens "Meisterverbund" (Österreichweit) als Firmen- und News-Portal für Meisterbetriebe.

TECH-STACK (Replit-friendly):
- Next.js (App Router) + TypeScript
- TailwindCSS
- Prisma ORM
- SQLite (lokal in Replit)
- Auth: eigene Email+Passwort Auth (bcrypt hashing, JWT oder Session-Cookie)
- Rollen: "admin" und "user"

DESIGN / BRANDING:
- Stil: seriös, handwerklich, Wappen/Meister-Siegel Look wie das mitgelieferte Logo (Bordeaux/Weiß, klare Serif-Headline, moderne Sans für Text).
- Farbpalette: Dunkel-Bordeaux (#721110), Bordeaux (#AE191C), Off-White (#FDFBF9), Hellgrau (#D3B5B4).
- Lege das Logo als /public/logo.png ab und nutze es im Header links + als Favicon (falls möglich).
- UI: responsive, sauber, schnelle Ladezeiten, SEO-freundliche Seiten.

FUNKTIONEN (MUSS):
1) Öffentliche Seiten:
- Startseite: Hero mit Logo + Claim ("Meisterbetriebe in Österreich – News, Bewertungen, Vertrauen")
- Sektion "Neueste News & Berichte" (nur Admin erstellt)
- Sektion "Top bewertete Meisterbetriebe" (aus Bewertungen berechnet)
- Firmenverzeichnis /firmen: Suche + Filter (Bundesland, Gewerke/Kategorie)
- Firmendetail /firmen/[id]:
  - Firmenprofil (Name, Beschreibung, Adresse, Bundesland, Website/Telefon)
  - Sterne-Durchschnitt (0–5) + Anzahl Bewertungen
  - Liste Bewertungen (Kommentar + Sterne + Datum + Nutzername/alias)

2) Accounts:
- Registrierung/Login mit Email + Passwort
- User-Profilseite: eigene Bewertungen anzeigen und optional bearbeiten
- Nur eingeloggte User dürfen:
  - Sterne 0–5 vergeben
  - Kommentar schreiben
- Regel: 1 Bewertung pro User pro Firma (bei neuer Bewertung: upsert/aktualisieren)

3) Admin (nur Admin darf News/Berichte schreiben):
- Admin-Konsole unter /admin (geschützt, nur role=admin)
- Admin kann:
  - News/Berichte erstellen, bearbeiten, veröffentlichen (Markdown Editor reicht)
  - Firmen anlegen/bearbeiten (damit Meisterbetriebe gelistet werden)
  - Bewertungen/Kommentare löschen (soft delete)
  - Benutzer sperren (blocked=true), gesperrte User können nicht bewerten/kommentieren
- Admin Login:
  - Erzeuge initialen Admin NICHT im Code hardcoden!
  - Nutze ENV Variablen:
    ADMIN_EMAIL
    ADMIN_PASSWORD
  - Beim ersten Start soll ein Seed-Script laufen (oder ein Button/CLI), das den Admin anlegt, Passwort mit bcrypt hasht und role='admin' setzt.

DATENBANK (Prisma Schema):
- User: id, email (unique), passwordHash, role ('admin'|'user'), blocked (bool), createdAt
- Company: id, name, description, category, address, city, state (Bundesland), website, phone, isMasterVerified (bool), createdAt, updatedAt
- Review: id, userId, companyId, stars (int 0-5), comment (text), deleted (bool), createdAt, updatedAt
- Post: id, title, slug (unique), contentMarkdown, type ('news'|'bericht'), published (bool), createdAt, updatedAt, authorId (admin)

BERECHNUNGEN:
- Firmenrating: average(stars) nur von Reviews mit deleted=false
- Zeige auf Firmenseite Sterne als Icons.

SICHERHEIT / REGELN:
- Passwörter immer bcrypt (min. 10 rounds)
- Admin-Bereich und Admin-API Endpoints strikt per role prüfen
- Input Validation: Sterne nur 0..5, Kommentar max 1.000 Zeichen
- Rate-Limit light (optional) für Login/Review POST

ROUTEN / API:
- /api/auth/register, /api/auth/login, /api/auth/logout, /api/auth/me
- /api/companies (GET, admin POST/PUT)
- /api/companies/[id] (GET)
- /api/reviews (POST/PUT upsert), /api/reviews?companyId=...
- /api/admin/users (GET, PATCH block/unblock)
- /api/admin/reviews (DELETE soft delete)
- /api/posts (GET published), /api/admin/posts (CRUD)

SEITEN:
- / (Home)
- /firmen (Directory)
- /firmen/[id] (Detail)
- /news (Liste veröffentlichter Posts)
- /news/[slug] (Post Detail)
- /login /register
- /admin (Dashboard: Tabs für Posts, Firmen, Bewertungen, Benutzer)
